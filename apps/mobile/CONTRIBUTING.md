# Contributing Guide

Terima kasih telah berkontribusi ke DonasiTrack! Berikut adalah panduan kontribusi.

## Cara Berkontribusi

1. **Fork repository** di GitHub
2. **Clone** repository Anda: `git clone https://github.com/YOUR_USERNAME/DonasiTrack.git`
3. **Buat branch baru**: `git checkout -b feature/your-feature-name`
4. **Lakukan perubahan** dan commit dengan pesan yang jelas
5. **Push ke branch**: `git push origin feature/your-feature-name`
6. **Buat Pull Request** di GitHub

## Commit Message Convention

Gunakan conventional commits:

```
feat: add new feature
fix: fix a bug
docs: update documentation
style: update code style
refactor: refactor code
test: add tests
chore: maintenance tasks
```

Contoh:
```
feat: add user authentication
fix: resolve login validation error
docs: add architecture guide
```

## Code Style

- Gunakan `dart format` untuk format code
- Follow Dart naming conventions (camelCase untuk variables/functions, PascalCase untuk classes)
- Tambahkan comments untuk logic yang kompleks

## Pull Request Checklist

- [ ] Code telah di-test
- [ ] Tidak ada breaking changes
- [ ] Documentation sudah diupdate jika diperlukan
- [ ] Commit messages mengikuti convention
- [ ] Tidak ada unused imports atau variables

## Reporting Bugs

Jika menemukan bug:
1. Cek apakah bug sudah dilaporkan
2. Buat issue baru dengan detail:
   - Deskripsi bug
   - Steps to reproduce
   - Expected behavior
   - Actual behavior
   - Screenshots jika relevan

## Feature Requests

Untuk feature requests:
1. Cek apakah sudah ada discussion
2. Buat issue baru dengan:
   - Deskripsi feature
   - Use case
   - Contoh implementasi jika ada

## Questions?

Buka discussion di GitHub atau hubungi maintainers.

Terima kasih atas kontribusi Anda! 🙏
